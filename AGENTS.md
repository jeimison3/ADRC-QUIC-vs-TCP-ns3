# AGENTS.md

## Build system

- **ns-3 version**: 3.47
- **Build tool**: CMake (`./ns3 configure`, `./ns3 build`)
- **Scratch directory**: All `.cc` files in `scratch/` are auto-detected and compiled by CMake.
  Each file with a `main()` function becomes a standalone executable.
- **Build command**: `./ns3 build` (from `/home/jeimison/Documentos/Tools/ns-allinone-3.47/ns-3.47`)
- **Compile single scratch**: `./ns3 build scratch/<filename>.cc`
- **Run**: `./ns3 run "<executable-name> --args"` — note: executable prefix is `scratch_` automatically,
  but `./ns3 run` handles name resolution (use without prefix, e.g., `tcp-udp-comparison`).

## TCP connection state traces

**`State` TracedSource** on `TcpSocketBase` tracks the full TCP state machine
(`TcpStates_t` enum: `CLOSED` → `SYN_SENT` → `SYN_RCVD` → `ESTABLISHED` →
`CLOSE_WAIT` → `LAST_ACK` → `FIN_WAIT_1` → `FIN_WAIT_2` → `CLOSING` → `TIME_WAIT`).

Config path:
```
/NodeList/<id>/$ns3::TcpL4Protocol/SocketList/<idx>/State
```

Callback signature:
```cpp
void (*)(TcpSocket::TcpStates_t oldState, TcpSocket::TcpStates_t newState)
```

**Detecting handshake success**: transition to `ESTABLISHED` (value 4) = 3-way handshake completed.
**Detecting handshake failure**: socket stays in `SYN_SENT` (value 2) then goes to `CLOSED` or timeout.

**`CongState` TracedSource** (`TcpCongState_t`): `CA_OPEN` = normal operation (connection alive).

### Socket-level callbacks (per-socket, no Config path)

```cpp
Ptr<Socket> sock = Socket::CreateSocket(node, TcpSocketFactory::GetTypeId());
sock->SetConnectCallback(
    MakeCallback(&OnConnectionSucceeded),   // void(Ptr<Socket>)
    MakeCallback(&OnConnectionFailed));     // void(Ptr<Socket>)
sock->SetCloseCallbacks(
    MakeCallback(&OnNormalClose),           // void(Ptr<Socket>)
    MakeCallback(&OnErrorClose));           // void(Ptr<Socket>)
```

## QUIC module: handshake detection

**Critical limitation**: `QuicSocketBase::m_socketState` is a `TracedValue<QuicStates_t>` but the
`"State"` TraceSource is **commented out** in `GetTypeId()` (`quic-socket-base.cc:242-245`).
Workarounds:

### Strategy A: Rx trace + header type inspection (most precise)
```cpp
void RxTrace(Ptr<const Packet> p, const QuicHeader& h, Ptr<const QuicSocketBase> s) {
    if (h.IsInitial())     { /* INITIAL packet (handshake start) */ }
    if (h.IsHandshake())   { /* HANDSHAKE packet (server response) */ }
    if (h.IsORTT())        { /* 0-RTT Protected data */ }
    if (h.IsVersionNegotiation()) { /* Version negotiation */ }
    if (h.IsShort())       { /* Short header (post-handshake data) */ }
}
```
Config path:
```
/NodeList/<id>/$ns3::QuicL4Protocol/SocketList/*/QuicSocketBase/Rx
```

### Strategy B: CongState → CA_OPEN
When handshake completes, `ConnectionSucceeded()` sets congestion state to `CA_OPEN`.
Config path:
```
/NodeList/<id>/$ns3::QuicL4Protocol/SocketList/<idx>/QuicSocketBase/CongState
```

### Strategy C: CongestionWindow first growth
After handshake, cWnd starts growing from initial window.
Config path:
```
/NodeList/<id>/$ns3::QuicL4Protocol/SocketList/<idx>/QuicSocketBase/CongestionWindow
```

### QUIC 0-RTT support

- **Enable**: `QuicL4Protocol` attribute `"0RTT-Handshake"` (Boolean, default `false`).
  Set via: `Config::SetDefault("ns3::QuicL4Protocol::0RTT-Handshake", BooleanValue(true));`
- **Detection**: via Rx trace: `header.IsORTT()` returns `true` for 0-RTT packets.
- **Fast path**: `DoFastConnect()` sets state directly to `OPEN` (skips CONNECTING_CLT).

### QUIC connection state enum (`QuicStates_t`)
```
IDLE = 0, LISTENING = 1, CONNECTING_SVR = 2, CONNECTING_CLT = 3, OPEN = 4, CLOSING = 5
```

### Available QUIC TracedSources (all registered in GetTypeId)
```
RTO, RTT, NextTxSequence, HighestSequence, CongState, CongestionWindow,
SlowStartThreshold, Tx, Rx   (+ BbrState if using TcpBbr)
```
NOT available: `State`, `AdvWND`, `RWND`, `BytesInFlight`, `HighestRxSequence`, `HighestRxAck`

## Measuring connection success rate

### TCP approach
```
1. Create a socket with TcpSocketFactory
2. Register SetConnectCallback(successFn, failFn) + SetCloseCallbacks(normalFn, errorFn)
3. Call sock->Connect(remoteAddr)
4. On success callback: record latency (now - connectTime), increment success counter
5. On fail/error callback: increment failure counter
6. Report: successRate = successes / (successes + failures), avgHandshakeTime
```

### QUIC approach
```
1. Hook into "Rx" trace on QuicSocketBase
2. When IsInitial() detected: record start time for that socket
3. When IsHandshake() detected + CongState becomes CA_OPEN: record handshake complete
4. For 0-RTT: when IsORTT() detected: mark as 0-RTT flow
5. Use a timeout: if no IsHandshake() or CA_OPEN within N seconds → failure
```

### UDP (baseline)
```
UDP has no handshake. Measure: first packet received vs expected first packet.
Success = any packet received within timeout window.
```

## Short-lived connection pattern

For success-rate scripts, each client node runs multiple short connections:
```cpp
// Schedule recurring connection attempts
void ScheduleConnection(Ptr<Node> client, Ipv4Address serverAddr, uint16_t port,
                        uint32_t remaining, Time interval, Time timeout) {
    if (remaining == 0) return;
    Ptr<Socket> sock = Socket::CreateSocket(client, TcpSocketFactory::GetTypeId());
    sock->SetConnectCallback(
        MakeCallback(&ConnSuccess),
        MakeCallback(&ConnFailure));
    sock->Connect(InetSocketAddress(serverAddr, port));
    Simulator::Schedule(timeout, &ForceTimeout, sock);
    Simulator::Schedule(interval, &ScheduleConnection,
                        client, serverAddr, port, remaining - 1, interval, timeout);
}
```
